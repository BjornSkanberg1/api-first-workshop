Quick Start

This section explains the minimum set of changes required to run the demo on the VM. For further information on configuring and customising the demo, see User Guide.

Initial Setup Configuration

When you first create the VM Blueprint, you need to configure the demo to use the correct settings. This can be done by:

  1. Use the Remote Desktop Connection to log into the VM using your SoftwareAG credentials.
  2. Double-click on the ‘Start Services.bat’ script on the Desktop, wait about 2 minutes for the products to start. (This will set the services to automatic and the next startup should be quicker). Users should only need to perform this step once after creating the VM instance.

Opening the Home page

Double click on the ‘Demo Home’ shortcut on the public Desktop to open the demo Home page. Alternatively navigate to the following URL using your internet browser (preferably Google Chrome).

http://sagbase.eur.ad.sag:8585/ConnectedServices#!/home

In either case, you may be asked for the following credentials:

  • Username = Administrator
  • Password = manage

Starting the Demo

To start the demo:

  1. Click the ‘Start Demo’ button on the ‘Home’ page to start the Demo.

  2. The Demo will be running shortly after you see the message ‘Demo Started’.

    Note: In some cases, there may be a short delay after the message is displayed before the demo is fully operational. This is expected, and is typically due to the creation of historic data, creation of simulators, etc.

  3. Open the demo tab by selecting it from the top of the screen, or from this link:
    http://sagbase.eur.ad.sag:8585/ConnectedServices#!/main

Stopping the Demo

The demo can be stopped by clicking the ‘Stop Demo’ button on ‘Home’ page.

Note: The data from the demo will be persisted after stopping the demo, and will be re-loaded after the demo is restarted.

Accessing the Demo remotely

To access the demo remotely (using the web browser on your machine rather than inside the VM) you’ll need to make a change to your hosts file.

  1. Read the IP address of the VM. This can be done by :
    - Open a command prompt on the VM
    - Enter ipconfig
    - Copy the IPV4 Address

  2. On your local machine update the hosts file. This can be done by:
    - Opening the C:\Windows\System32\drivers\etc\hostsfile in a text editor.
    - Adding the following new line:
    <ip-address> sagbase.eur.ad.sag #Demo Machine
    where ip-address is the IPV4 Address you copied.
    Note: The user needs to have Administrator access to change the hosts file.
